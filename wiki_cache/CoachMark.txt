## 개발 사양

## 규칙
*(UX 규칙 작성. 아래는 예시)*
**화살표 방향 및 배치**
- 코치마크의 화살표는 상/하/좌/우 네 가지 방향 중 하나를 반드시 선택하며, 반드시 표시해야 함

- 화살표 인디케이터의 위치를 한 쪽 변의 시작/중앙/끝을 선택하여 표시할 수 있음

 [image] [image]
**배치**
- 코치마크는 해당 콘텐츠 주변에 배치

- 코치마크가 가리키는 콘텐츠의 인터랙션 영역을 가리지 않도록 배치

[image][image]**크기**
- 코치마크에서 지정한 너비를 사용하며 임의로 늘리거나 줄이지 않음

- 높이는 텍스트 분량에 따라서 변경 가능

[image]**내용 작성**
- 제목
가능한 1~3단어 이하의 간결한 용어를 사용하고, 자세한 설명은 본문에서 설명

- 화살표 인디케이터가 가리키는 기능의 명칭을 넣는 것을 추천

- 본문
본문은 완전한 문장으로 표시하며 과도하게 짧거나 긴 문장을 표시하지 않는다.

- 기능의 개요에 대해 유용한 정보를 최대한 간략하게 전달한다.

**인터랙션**
- 기본 동작
최초 런칭 시 코치마크가 노출됨

- 확인 버튼 클릭 시, 이후 해당 페이지 진입 시 코치마크는 다시 뜨지 않음

- 확인하지 않은 경우, 해당 페이지 진입 시마다 계속 코치마크가 노출됨

- 코치마크가 표시되는 동안에는 해당 영역 외의 인터랙션은 차단됨

- 데이터 보존 기준
코치마크 확인 여부는 마지막 접속 기준으로 40일간 DB에 저장됨

- 마지막 접속 이후 40일간 로그인하지 않은 경우,

- 다음 접속 시에는 확인 여부가 무효화되어

- 코치마크가 다시 최초 상태로 노출됨

- 예외 및 복구 처리
코치마크 노출 중 서버 장애 등 문제 발생 시 확인 버튼으로 코치마크를 닫는 것은 가능함

- 단 확인 상태가 서버에 정상 전송되지 않은 경우, 로그인 후 코치마크가 다시 노출될 수 있음

- 서버 전송이 정상 완료되면 더 이상 노출되지 않음
