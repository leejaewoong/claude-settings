## 규칙
*(UX 규칙 작성. 아래는 예시)*

아이템 슬롯
- 미션에 대한 보상 아이템을 제공할 시, **규격화된 아이템 슬롯**을 활용하여 표시규격화된 아이템 슬롯은 진행 정보 UI 좌측 또는 우측에 배치할 수 있음
미션 내용
- 미션 타이틀과 설명은 둘 중 하나는 반드시 표기
- 미션 부여 번호는 생략 가능
- 미션 프로그레스 바는 무료/유료/완료/미완료 등 조건에 관계 없이 반드시 표기
- 미션 완료 또는 잠금 등 미션 내용을 보여줄 필요가 없거나 제약을 걸어야 할 경우에는 백그라운드 딤 처리 적용
기타
- 버튼, 툴팁 등 외부 요소는 얹어 쓰기

## Property
*(개발자가 참고해 사전 구현할 수 있도록 작성. 아래는 예시)*

명칭
 | 상태
 | 기본값
 | 설명
 | 
Title | - True
- False
 | True | 미션 타이틀 표기 여부
 | 
Description | - True
 | True | 미션 설명 표기 (필수)
 | 
MissionState | - Default
- Complete
- Locked
 | Default
 | 미션 진행도 바
- Default: 미션이 진행되는 기본 상태 (진척도 0~완료 직전까지)
- Complete: 미션이 완료된 상태 (진척도 100%)
- Locked: 미션 잠김 상태 - 미션 진행도 바 숨김 + 딤 처리
 | 
Interaction | - Default
- Hover
- Selected
 | Default
 | 미션 인터랙션 조건에 따른 백그라운드 상태
- Default: 기본 상태로, 미션과 인터랙션 및 진행 가능
- Hover: 미션 호버 상태
- Selected: 미션 선택 상태
- Dimmed: 미션 완료 혹은 잠김 등 특정 조건으로 인해 미션과 인터랙션 불가능
 |
