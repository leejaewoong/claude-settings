# Bash safelist for ~/.claude/hooks/bash_safelist_check.py
#
# One bare executable per line. Lines starting with `#` are comments.
# Multi-word entries like `git status` are matched after the hook resolves
# git/sudo prefixes — see hooks/bash_safelist_check.py for the resolver.
#
# If every atomic executable in a bash invocation appears in this file,
# the hook auto-approves and the user gets no permission prompt. If even
# one atom is missing, the hook falls through silently and Claude Code's
# normal permission flow runs (showing the prompt to the user).
#
# Pure shell-syntax helpers (`[`, `[[`, `:`, `true`, `false`, `test`) and
# control keywords (`for`, `while`, `if`, etc.) are handled transparently
# by the hook and do NOT need entries here.
