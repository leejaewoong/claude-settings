# 컨셉 B: 경쟁작 레이아웃 참조

```yaml
컨셉: B
이름: "경쟁작 레이아웃 참조"
설명: "경쟁 게임 UI를 참고하여 PUBG DS 위에 경쟁작의 레이아웃/시각 요소를 오버레이한 변형 시안"
스크린샷_분석: false    # PUBG 스크린샷 대신 경쟁작 패턴 참조
DS_토큰_사용: base      # 기본 토큰 유지, accent/비율은 경쟁작 규칙으로 오버라이드
```

> **폰트 통일**: 모든 컨셉 공통으로 `PUBG Body` (본문) / `PUBG Headline` (제목) 사용 — 경쟁작 타이포 스타일은 참조하지 않음

---

## 참조 안내 의무

컨셉 B로 생성 시, 사용자에게 **반드시** 참조한 경쟁작 정보를 안내한다:

```yaml
안내_형식: |
  컨셉 B 시안입니다.
  참조: {경쟁작명} — {해당 화면 유형}
  주요 적용: {적용된 시각 요소 1-2줄 요약}
```

---

## 화면별 추천 경쟁작 매핑

Phase 1에서 확정된 메뉴 경로를 기반으로 가장 적합한 경쟁작을 자동 선택한다.

    이유: "데이터 시각화 + 매치 히스토리 표현이 정교"

## 공통 오버라이드 규칙

> **색상 참조**: config.md COLOR 섹션 — 경쟁작의 레이아웃과 시각 요소만 참조하고, 색상 팔레트는 PUBG DS를 사용한다.

경쟁작 참조 시에도 아래 PUBG DS 기본은 유지한다:

```yaml
유지_항목:
  폰트: "PUBG Body / PUBG Headline — 모든 컨셉 공통, 변경 금지"
  해상도: "1920x1080 고정"
  GNB_Footer: "프리셋 그대로 사용"
  컴포넌트_크기: "DS 컴포넌트 크기 규격 유지 (SquareButton, ItemSlot 등)"
  Mock_데이터: "동일 데이터셋"
  색상_팔레트: "config.md COLOR 섹션 참조 — var(--text-primary), var(--text-secondary) 등 PUBG DS 팔레트 사용"

오버라이드_항목:
  배경_처리: "경쟁작의 패널 투명도/배경 스타일 적용"
  레이아웃_비율: "경쟁작의 구조 성향 적용 (카드 크기, 여백 비율, 밀도)"
  시각_요소: "경쟁작의 고유 시각 장식 적용 (클립패스, 글로우, 텍스처)"
  border_radius: "경쟁작 성향에 따라 4px~20px 범위 조정"
```

## 셀프 체크리스트

```yaml
셀프_체크:
  - "[ ] 참조 경쟁작 안내 메시지가 사용자에게 표시되었는가?"
  - "[ ] 폰트가 PUBG Body/Headline으로 통일되었는가? (외부 폰트 없음)"
  - "[ ] 색상이 config.md COLOR 섹션의 PUBG DS 팔레트를 따르는가?"
  - "[ ] 경쟁작의 고유 시각 요소가 1개 이상 반영되었는가?"
  - "[ ] GNB/Footer 프리셋이 변형 없이 사용되었는가?"
  - "[ ] DS 컴포넌트 크기 규격이 유지되었는가?"
  - "[ ] PUBG와 시각적으로 구분 가능한가?"
```
