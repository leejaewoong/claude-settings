# Phase 1: 대화로 화면 확정

> **필수 참조**: `commands/figma/menu-tree.md` (메뉴 구조), `commands/figma/layout-components.md` (GNB/LNB/FOOTER)

**규칙:**
- **한 번에 하나의 질문만** 제시하고, 사용자 응답을 받은 뒤 다음 질문으로 진행한다
- 여러 질문을 한 메시지에 묶어서 보내지 않는다
- 매 질문마다 선택지를 번호 목록으로 제시한다
- 선택지 마지막에는 항상 **"N. 신규 화면"** 을 포함한다
- 사용자가 번호 또는 텍스트로 답하면 다음 뎁스로 진행한다
- 더 이상 하위 뎁스가 없거나 사용자가 "이걸로 확정"하면 Phase 2로 넘어간다
- 신규 화면 선택 시 화면 이름과 목적을 자유 입력 받는다

**제작 제외 화면:**
- **미리보기 화면** (아이템 미리보기, 세트 미리보기 등)과 **팝업 화면** (구매 확인, 필터 등)은 제작하지 않는다
- 사유: 기존 프로덕트의 미리보기/팝업 규칙이 정교하여 별도 시안이 불필요
- **단, 해당 화면으로 진입하는 버튼/트리거는 이전 화면(부모 화면)에 반드시 포함해야 한다**
  - 예: 아이템 카드 클릭 → 미리보기 진입이면, 아이템 카드는 부모 화면에 표시
  - 예: "구매" 버튼 클릭 → 구매 확인 팝업이면, "구매" 버튼은 부모 화면에 표시
- 사용자가 미리보기/팝업을 선택하면 안내 후 부모 화면 제작을 권유한다

**Q4 — 추가 요구사항:**

```
화면에 대해 추가로 알려줄 내용이 있나요?
```

```yaml
위키_URL_처리:
  감지_패턴: "atlassian.net/wiki/ 또는 confluence 도메인이 포함된 URL"
  절차:
    1_env_로드:
      파일: "C:\\Users\\jaewoong\\.claude\\.env"
      필수_변수:
        - CONFLUENCE_EMAIL
        - CONFLUENCE_API_TOKEN
        - CONFLUENCE_BASE_URL
      실패시: ".env 파일에 Confluence 인증 정보가 없습니다. 텍스트 입력으로 진행합니다."

    2_pageId_추출:
      패턴_1: "/pages/{pageId}/"
      패턴_2: "/pages/{pageId}/{title}"
      패턴_3: "pageId={pageId}"
      예시: "https://xxx.atlassian.net/wiki/spaces/SPACE/pages/123456/제목 → pageId=123456"

    3_API_호출:
      주의: "WebFetch 도구 사용 금지 — 인증 헤더 미지원으로 401 에러 발생. 반드시 curl -u 사용"
      명령: |
        source C:/Users/jaewoong/.claude/.env
        curl -s \
          -u "${CONFLUENCE_EMAIL}:${CONFLUENCE_API_TOKEN}" \
          -H "Content-Type: application/json" \
          "${CONFLUENCE_BASE_URL}wiki/rest/api/content/{pageId}?expand=body.storage,version"
      실패시: "위키 조회에 실패했습니다. 텍스트로 직접 입력해주세요."

    4_내용_활용:
      - "응답의 body.storage.value (XHTML)에서 텍스트 추출"
      - "기획 의도, UI 구성, 데이터 구조, 비즈니스 규칙 등 설계 지침 파악"
      - "파악한 내용을 Q4 추가 요구사항으로 자동 반영"
      - "사용자에게 요약 표시: '위키에서 다음 설계 지침을 확인했습니다: ...'"
```

**로비/캐릭터 선택 처리:**
```yaml
선택_처리:
  로비_포함시:
    참조_파일:
      - "SCREENSHOTS_DIR/로비.png — 눈 내리는 풍경 (메인 로비)"
      - "SCREENSHOTS_DIR/커스터마이즈_로비.png — 커스터마이즈 로비 배경"
    배치: "HTML 배경 이미지 또는 배경 레이어로 삽입"

  캐릭터_포함시:
    참조_파일:
      - "SCREENSHOTS_DIR/캐릭터 모델링.png — 전신 캐릭터 (검은 배경)"
      - "SCREENSHOTS_DIR/미리보기_캐릭터_단품.png — 단품 미리보기 레이아웃 참조"
      - "SCREENSHOTS_DIR/미리보기_캐릭터_세트.png — 세트 미리보기 레이아웃 참조"

    배치_규칙:
      Y좌표: |
        캐릭터 발 밑이 Footer 바로 윗부분에 오도록 고정한다.
        Footer와의 간격은 0~50px 이내 (최대 50px 띄울 수 있음).
        계산: bottom = Footer높이(56px) + 간격(0~50px)
      X좌표: |
        자유롭게 변경 가능.
        단, 캐릭터 본체가 화면 밖으로 잘리면 안 된다.
        검은 배경 부분은 잘려도 무방하다.
      크기: |
        캐릭터 높이를 850px로 고정한다 (height: 850px, width: auto).
        캐릭터 위에 UI/컴포넌트가 표시될 수 있으므로 z-index를 낮게(1) 설정한다.

    스타일_처리:
      배경_검은색: |
        캐릭터 이미지의 검은 배경 가장자리가 페이지와 자연스럽게 블렌딩되도록
        .layout에 has-character 클래스를 추가하고 배경을 #111115으로 강제한다.
        .layout.has-character { background: #111115; }
        .layout.has-character .content { background: #111115; }
      배경_제거: "캐릭터 모델링.png의 검은 배경은 mix-blend-mode: lighten으로 제거 (배경 #111115 + lighten 블렌딩 — #000~#333 검은 영역이 #111115보다 어두워 정상 제거)"
      그림자: "캐릭터 발 아래 원형 그림자(radial-gradient) 추가 권장 (z-index: 0)"
      레이어: "캐릭터는 z-index: 1, UI 패널은 z-index: 3+, GNB는 z-index: 10"

    적용_화면_예시:
      - "커스터마이즈 (옷장/외형/포즈) — 캐릭터 중앙~우측, 좌측에 아이템 패널"
      - "상점 미리보기 — 캐릭터 중앙, 좌측에 상품 정보"
      - "로비 — 배경 위 캐릭터 중앙 배치"
```

**Q7 — 화면 분할 수:**

```
기획하신 컨셉을 몇 개 화면으로 나누어 제작할까요?

1. 단일 화면 — 하나의 1920×1080 프레임
2. 2~3개 화면 — 메인 + 상세 등 소규모 구성
3. 3~5개 화면 — 다양한 상태/플로우를 포함하는 중규모 구성
A. 자동 판단 — 화면 특성에 따라 Claude가 결정
```

**Q7 자동 판단 로직 (A 선택 시):**
```yaml
Q7_자동판단:
  설명: "메뉴 뎁스, 하위 항목 수, Q4 요구사항을 종합하여 판단"

  단일_화면:
    조건:
      - "3뎁스 이상의 말단 화면 (더 이상 하위 분기 없음)"

  2~3개_화면:
    조건:
      - "2뎁스 화면으로 하위 항목이 2~3개인 경우"

      - "Q4에서 1~2개의 추가 상태/뷰 언급"
    해당_예시: "상점 > 아이템, 패스 > 서바이버 패스, 전적 > 매치 히스토리"

  3~5개_화면:
    조건:
      - "2뎁스 화면으로 하위 항목이 4개 이상인 경우"
      - "복수 상태 전환이 필요한 플로우"
      - "Q4에서 여러 상태/단계/시나리오 언급"
      - "인게임 화면 (HUD + 맵 + 인벤토리 등 다수 뷰)"
    해당_예시: "커스터마이즈 > 캐릭터, 인게임 > HUD, 제작소 > 컨텐더"

  결정_후_안내: |
    사용자에게 판단 결과를 안내한다:
    "{N}개 화면으로 구성합니다. (사유: {판단 근거 1줄})"
```

**Q8 — 디자인 컨셉 선택:**

```
어떤 디자인 컨셉으로 생성할까요? (복수 선택 가능, 쉼표 구분)

1. PUBG 레이아웃 준수 — 기존 PUBG 레이아웃을 참고한 디자인
2. 경쟁작 레이아웃 참조 — 경쟁 게임 레이아웃을 참고한 디자인

4. 모두 (A+B+C) — 3가지 컨셉 모두 생성

예: A 또는 A,B 또는 D
```

**컨셉 선택 처리:**
```yaml
Q8_처리:
```

**레이아웃 선택 후 처리:**
```yaml
선택_처리:
  GNB_포함시:
    - GNB 탭 중 현재 메뉴(1뎁스)를 Selected 상태로 표시
    - 나머지 탭은 Normal 상태
  LNB_포함시:
    - 확정된 메뉴의 2뎁스 항목들을 LNB 탭으로 배치
    - 현재 선택된 2뎁스를 Selected 상태로 표시
  FOOTER_포함시:
    - 화면 목적에 맞는 버튼 구성 문의
    - "이 화면에 필요한 하단 버튼이 있나요? (예: 저장, 확인, 닫기 등)"
  프리셋_로드:
    - commands/figma/presets/ 의 프리셋 HTML 사용
    - GNB+LNB → gnb.html (1920×162px, 보조재화 자동 배치)
    - LNB Only → lnb.html (1920×162px)
    - FOOTER → footer.html (1920×56px)
    - Phase 4에서 동적 마커({{ACTIVE_TAB}}, {{LNB_TABS}} 등) 치환하여 삽입
```
