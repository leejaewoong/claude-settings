사용자가 인자로 넘긴 명령어 문자열에서 허용되지 않은 bash 명령어를 찾아 안내하고, 승인 시 허용 목록에 추가합니다.

## 입력

사용자가 `/allow-cmd` 뒤에 전달한 명령어 문자열 전체를 분석 대상으로 사용합니다.

예시: `/allow-cmd cd /project && find . -name "*.py" | grep test | head -20`

## 실행 절차

### 1단계: 명령어 파싱

인자로 받은 명령어 문자열에서 개별 명령어를 추출합니다:
- 파이프 `|`, AND `&&`, 세미콜론 `;`, OR `||` 등으로 분리
- 각 명령어의 **첫 번째 단어**(실행 파일명)만 추출
- 서브쉘 `$(...)`, 백틱 등 내부 명령어도 추출
- `cd`는 쉘 내장 명령이므로 제외

### 2단계: 허용 목록 비교

`~/.claude/settings.json`의 `permissions.allow` 배열에서 `Bash(...)` 패턴들을 읽습니다.

각 추출된 명령어에 대해:
- `Bash(<command>:*)` 또는 `Bash(<command>)` 패턴이 있으면 → **허용됨**
- git 하위 명령어는 `Bash(git <subcommand>:*)` 형태로 확인
- 패턴이 없으면 → **미허용**

### 3단계: 결과 안내

**모든 명령어가 허용된 경우:**
> "입력된 명령어는 모두 허용 목록에 포함되어 있습니다."

**미허용 명령어가 있는 경우**, 테이블로 안내합니다:

| 명령어 | 설명 | 상태 |
|--------|------|------|
| `find` | 파일/디렉토리 검색 | 허용됨 |
| `grep` | 정규표현식 패턴 검색 | **미허용** |
| `awk` | 텍스트 처리 및 패턴 매칭 도구 | **미허용** |

- 설명은 한국어로 간결하게 작성
- 이미 허용된 명령어도 함께 표시하여 전체 현황 파악 가능하게 함

### 4단계: 사용자 확인

AskUserQuestion을 사용하여 미허용 명령어의 추가 여부를 확인합니다:
- 옵션: "모두 추가", "선택적 추가", "취소"
- "선택적 추가" 선택 시 multiSelect로 개별 명령어 선택 가능

### 5단계: 파일 업데이트

승인된 명령어에 대해 두 파일을 업데이트합니다:

**settings.json 업데이트:**
- `permissions.allow` 배열에 `"Bash(<command>:*)"` 항목 추가
- 기존 시스템 유틸리티 명령어들(`Bash(pwd)`, `Bash(which:*)` 등) 근처에 삽입

**settings-reference.md 업데이트:**
- "시스템 유틸리티" 섹션의 테이블에 새 행 추가
- 형식: `| \`Bash(<command>:*)\` | 한국어 설명 |`

### 6단계: 커밋 및 푸시

파일 업데이트 완료 후 변경사항을 커밋하고 원격 저장소에 푸시합니다:

1. `git add settings.json settings-reference.md`
2. 커밋 메시지는 `/commit` 커맨드 규칙을 따름
   - 예시: `🔨Env(settings.json, settings-reference.md): <추가된 명령어> 허용 명령어 추가`
3. `git push`로 원격 저장소에 반영

## 주의사항

- 파일명, 명령어명은 영어 유지
- 설명과 안내 메시지는 한국어로 작성
- `git rm --cached` 같은 git 하위 명령어는 `Bash(git rm:*)` 형태로 확인
- 이미 허용된 명령어는 중복 추가하지 않음
